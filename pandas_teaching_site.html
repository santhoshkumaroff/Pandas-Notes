03 Column Selection — Series vs DataFrame Core Concept

Single Column → Returns a Series

Use single brackets df['column_name']. Result is a Pandas Series — a 1-dimensional labelled array (a single column with an index).

04 Row Selection — iloc vs loc Must Know
🧠
Memory Trick
iloc → integer position   |   loc → label (name). This distinction is everything.

iloc — Position-Based Indexing

Works like Python list indexing — uses integers (0, 1, 2…). The slice end is exclusive (just like Python).

python
# Single row at position 0 (first row)
df.iloc[0]

# Rows 1 to 4 (end is EXCLUSIVE — row 5 not included)
df.iloc[1:5]

# All rows, first 3 columns only (: means all rows)
df.iloc[:, 0:3]

# Specific rows: row 0 and row 3
df.iloc[[0, 3]]
▸ df.iloc[0] Output
gender female race/ethnicity group B parental level of education bachelor's degree lunch standard test preparation course none math score 72 reading score 72 writing score 74 Name: 0, dtype: object

loc — Label-Based Indexing

Uses row/column labels (names). With the default index, labels look like integers — but the key difference: loc's slice end is INCLUSIVE.

python
# Single row by label
df.loc[0]

# Rows 0 and 3 (list of labels)
df.loc[[0, 3]]

# All rows, only 'gender' column
df.loc[:, ['gender']]

# Rows 10 onwards, columns 'gender' TO 'lunch' (INCLUSIVE end!)
df.loc[10:, 'gender':'lunch']
▸ df.loc[10:, 'gender':'lunch'] Output (first 3 rows)
gender race/ethnicity parental level of education lunch 10 female group C associate's degree standard 11 male group D some college free/reduced 12 male group A high school free/reduced

⚠️
Common Mistake
Beginners mix up loc and iloc. Remember: df.iloc[0:5] returns 5 rows (0,1,2,3,4). df.loc[0:5] returns 6 rows (0,1,2,3,4,5) — because loc is inclusive!
05 — Filtering

Boolean Indexing & Filtering

Filtering is how you ask: "Give me only the rows that match this condition." It's one of the most powerful and frequently-used Pandas skills.

05 Filtering / Boolean Indexing High Frequency
1
Create a Boolean Mask
A comparison like df['gender'] == 'female' returns a Series of True/False values — one per row.
2
Apply the Mask
Pass the mask inside df[mask]. Pandas keeps only rows where the value is True.
3
Combine Conditions
Use & for AND, | for OR — always wrap each condition in parentheses.

Step 1 — The Boolean Mask

python
# This creates a True/False Series
mask = df['gender'] == 'female'
print(mask.head())
▸ Output
0 True 1 True 2 True 3 False 4 False Name: gender, dtype: bool

Step 2 — Apply the Mask

python
# Keep only female students
female_df = df[df['gender'] == 'female']
print("Female students:", len(female_df))
female_df.head(3)
▸ Output
Female students: 518

Step 3 — Multiple Conditions

python
# AND: female students with math score > 70
result1 = df[(df['gender'] == 'female') & (df['math score'] > 70)]
print("Female, math > 70:", len(result1))

# AND: male students with standard lunch
result2 = df[(df['gender'] == 'male') & (df['lunch'] == 'standard')]
print("Male, standard lunch:", len(result2))

# OR: female OR math score > 90
result3 = df[(df['gender'] == 'female') | (df['math score'] > 90)]
print("Female OR math > 90:", len(result3))
▸ Output
Female, math > 70: 253 Male, standard lunch: 352 Female OR math > 90: 539

Why Parentheses Are Critical

python
# ❌ WRONG — Python errors because & has higher precedence
# df[df['gender']=='female' & df['math score']>70]

# ✅ CORRECT — each condition wrapped in parentheses
df[(df['gender'] == 'female') & (df['math score'] > 70)]
⚠️
Operator Precedence
In Python, & and | bind tighter than ==, >, etc. Without parentheses, Python tries to AND the raw strings/numbers first and crashes with a TypeError. Always parenthesise!
06 — Categorical Analysis

Exploring Categorical Data

Categorical columns contain limited, repeated text values. Three Pandas functions let you explore them in seconds.

06 unique(), nunique(), value_counts() EDA Tools

unique() — What distinct values exist?

Real-world use: Spot typos and inconsistencies ('Male' vs 'male' vs 'M'). If unexpected values appear, data needs cleaning.

python
# Get all unique values in the gender column
print(df['gender'].unique())
▸ Output
['female' 'male']

nunique() — How many unique values?

Real-world use: Understand cardinality. A column with 900 unique values out of 1000 rows might be an ID — not a category!

python
# Count how many unique values each column has
print("gender:", df['gender'].nunique())
print("race/ethnicity:", df['race/ethnicity'].nunique())
print("parental education:", df['parental level of education'].nunique())
▸ Output
gender: 2 race/ethnicity: 5 parental education: 6

value_counts() — How frequent is each value?

Real-world use: Detect class imbalance before training a model. If one category dominates, your model may be biased.

python
# Count students per gender
print(df['gender'].value_counts())

# Show as percentage
print(df['gender'].value_counts(normalize=True).round(2))
▸ Output
female 518 male 482 Name: gender, dtype: int64 female 0.52 male 0.48 Name: gender, dtype: float64

Loop Through All Categorical Columns

A powerful pattern for quickly auditing all categorical columns in one block of code.

python
# Define all categorical columns
cat_cols = [
    'gender',
    'race/ethnicity',
    'parental level of education',
    'lunch',
    'test preparation course'
]

# Loop: print unique values for each column
for col in cat_cols:
    print(f"\n--- {col} ---")
    print(df[col].unique())
▸ Output
--- gender --- ['female' 'male'] --- race/ethnicity --- ['group B' 'group C' 'group A' 'group D' 'group E'] --- parental level of education --- ["bachelor's degree" 'some college' "master's degree" "associate's degree" 'high school' 'some high school'] --- lunch --- ['standard' 'free/reduced'] --- test preparation course --- ['none' 'completed']
💡
Pro Tip
In real projects, always run this loop after loading new data. Look for unexpected values like 'N/A' stored as text, extra spaces (' male' vs 'male'), or mixed casing — these silently break analysis.
07 — Missing Values

Detecting Missing Data

Missing values are one of the most common data quality issues. Pandas makes detecting them straightforward.

07 isnull() and isnull().sum() Data Quality

Missing values appear as NaN (Not a Number) in Pandas. They arise from skipped survey questions, sensor failures, incomplete form fills, or bad database joins.

python
# Returns True/False for every cell
df.isnull().head(3)

# Count missing values per column — most useful check
print(df.isnull().sum())

# Total missing in entire dataset
print("Total missing:", df.isnull().sum().sum())

# Percentage missing per column
print((df.isnull().mean() * 100).round(2))
▸ Output
gender 0 race/ethnicity 0 parental level of education 0 lunch 0 test preparation course 0 math score 0 reading score 0 writing score 0 dtype: int64 Total missing: 0
✅
Great News
This dataset has zero missing values! In real projects though, always run df.isnull().sum() as part of your standard EDA routine before any analysis.

Interview Common Interview Questions
What is the difference between loc and iloc?
loc uses labels/names; iloc uses integer positions. loc's slice end is inclusive, iloc's is exclusive.
Series vs DataFrame — what's the difference?
A Series is 1D (one column). A DataFrame is 2D (rows + columns). Selecting one column gives a Series; selecting multiple gives a DataFrame.
How do you check for missing values?
Use df.isnull().sum() for count per column; df.isnull().sum().sum() for total across the dataset.
Why must you use parentheses with & and |?
Because & and | have higher operator precedence than ==, >, < in Python. Without parentheses, Python evaluates in the wrong order and throws a TypeError.
When would you use value_counts()?
To understand the distribution of a categorical column — e.g. checking class balance before training a machine learning model.
